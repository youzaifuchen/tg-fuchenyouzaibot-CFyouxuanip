在遇到的 $'\r': command not found 和 syntax error near unexpected token 都是经典的 Windows 换行符问题。💡

详细分析：


1️⃣ 原因

你在 Windows 上编辑或上传脚本，GitHub 上的文件带了 CRLF (\r\n) 换行。

Linux / Termux / Bash 只认 LF (\n) 换行。

所以每行末尾多了 \r，Bash 解析不了就报 $'\r': command not found。



---

2️⃣ 解决办法

方法一：用 dos2unix 转换

在 Termux 或 Linux 上：

pkg install dos2unix -y   # 如果没装
dos2unix ym.sh
bash ym.sh


---

方法二：用 sed 一行命令去掉 \r

sed -i 's/\r$//' ym.sh
bash ym.sh


---

方法三：直接在 curl 时用 Bash 处理（不保存文件）

curl -sSL https://raw.githubusercontent.com/fuchenyouzai/tg-fuchenyouzaibot-CFyouxuanip/main/ym.sh | sed 's/\r$//' | bash

> 这样就不用手动保存和转换了。
